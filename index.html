Research

In the summer of 2019, I did research at Carleton College through their SCSI program. Here are the results from across our 3 test batches:

Parallel Integration

Using the MPI Frame work, I built a parallel version of Euler Step. The dotted lines represent a Raspberry Pi CPU (with a 1.2 GHz clock speed) where as the straight line represents a Local Cluster CPU (with a 2 GHz clock speed)

The algorithm had a time complexity of n, and every successive core reduced the time to compute linearly. This was because each core could work individually on each batch, as all the data was independent of each other.

Conclusion: it's important to note that, the speed of each core and the number of cores are semi-interchangeable, and also "equally" important to time complexity.

Conways Game of Life

Using the MPI Frame work, I built a parallel version of Conways Game of Life. The dotted lines represent a Raspberry Pi CPU (with a 1.2 GHz clock speed) where as the straight line represents a Local Cluster CPU (with a 2 GHz clock speed)

The algorithm had a time complexity of n^2. However, adding more cores after 4 would actually reduce the time complexity. This was because after every generation, each of the cores would have to communicate with each other, so the benefit of an extra core is less than the "cost" of communication.

Conclusion: when you have a program that has a lot of operations, but each operation has a low time complexity, adding more cores doesn't always help, but increasing the speed of the cores will always help with time complexity.

Waterfill Path Finding

Using the MPI Frame work, I built a parallel version of Waterfill Search. The dotted lines represent a Raspberry Pi CPU (with a 1.2 GHz clock speed) where as the straight line represents a Local Cluster CPU (with a 2 GHz clock speed)

The algorithm had a time complexity of n^2. It can be seen that adding more cores has a minimal effect on the time to compute, but the speed of each core has a large effect. This is because each "core" will only compute within its own zone (hard to explain in text), which means that much of the time cores are not working.
